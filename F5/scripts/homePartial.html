Web & Mobile Guidelines Introduction

The need for design guidelines

Developing software without a design first approach will cause user experience issues , inconsistency and can cause an additional 50% more developing work because not following design guidelines the application will never be finished design wise. Ex: Positioning an element multiple times during development playing with width , height and margins trying to obtain a visually pleasant aspect without respecting a standardized process.

Goal

Creating consistent UI , offering customers the best possible UX ( User Experience ). This manual will provide mainly general base guidelines that will create consistency around F5 IT`s developed software within our products or our clients products.
