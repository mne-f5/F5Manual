Efficient Input Forms

Efficient Forms

Given that F5 IT`s develops complex apps with data and forms , creating practical forms to ease users form completion quickly and without confusion is a must. Keypoins : Usability testing , field testing , eye tracking and user feedback.

Forms are still one of the most important type of interaction inside an app considered to be the final step towards completing the goal that the user is after.

Getting Started

Form Components

Typical forms are composed of five elements

Structure: It is defined by the ordering of the fields , page arrangement and connection between fields.

Input Field: Composed of text fields , password fields , radio buttons , sliders etc.

Label: Tied to a input field it tells the user what information the field needs.

Submit button: Action performing (submit data from fields.)

Feedback & Validation: Indicates to the user if the result is positive or negative (Success form completion / Incorrect data input). Validation checks in real time if data completion is valid.

Form Structure

Form are like conversations representing the communication between the user and the app.

Logical Ordering

Forms should be arranged in a logical manner - Don`t ask for a phone number before asking the name

Avoid unnecesary data

Avoid asking for extra information as it could stress the user if the forms are to long

Group related forms

Group fields that are related can be joined together making it easy for the user to scan the form.

One column always

Forms shouldn`t consist of more than one column as using multiple columns can create inconsistencies. Users would be required to scan in a Z pattern slowing speed of understanding. Yet single columns forms follow a single down the page path.

Number of fields

Always minimize the number of fields as much as possible. Try to combine multiple fields.

Label alignment

According to Matteo Penzo`s article on label placement claims that forms are completed faster if the label is on top of the fields. However other researches found no difference between left labels or top labels.

Top labels

The main advantage is that they can easily fit the graphic interface (specially for mobile screens).

Left aligned labels

Requires more horizontal space , provides a slower completion rate but it could be helpful because slowing completion could prevent user errors.

Conclusion & Usage

The top aligned labels should be used on small forms - up to 6 inputs (login form , sign-up form etc.) Left aligned labels should be used on bigger forms - Because it can help avoid errors.

Small improvements in forms cand lead to better user satisfaction when it comes to forms. Users are usually hesitant to form filling and this can be easily changed when testing and improving the forms.
