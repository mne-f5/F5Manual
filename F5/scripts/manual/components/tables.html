Table design

Table

Applications for industries will contain most of the time data tables. Making tables in a consistent manner , offering easy filtering and manipulation of data included is a must.

Guidelines

Table Structure

A data table contains a header row that lists column names and rows for data. Data tables should give the users the ability to manipulate and query data and it will be placed inside a container.

A basic data table should contain the following:

  • Table content - font size 1 em. Font color - 86% black opacity.
  • Column header - 0.85em. Font color - 54% black opacity
  • Text alignment - Left align for text columns . Right align for number columns.

A background should be displayed on a row if a user hovers over any part of the row.

Column Padding

Padding between columns should be of 56 pixels.

Responsiveness

Making tables responsive on the various screens available can usually be a challenge. There are 3 ways to handle responsive tables that have become standard.

Horizontal overflow

Gives users the ability to view the whole table on a smaller screen by making the first row fixed and scrolling left and right to view the rest of the rows. The issue with this table is that users might not notice that the table is scrollable.

Transitional

Reformating the table in a card type . This is helpful because it displays all the fields without scroll but with the downside that it eliminates field comparison and makes scanning of information slower.

Priority table

This type is obtained by hiding non important fields on mobile screens. It is the best solution if sufficient information can be provided in a 3 row table with click ability to show the rest of details.
