Defining a design system

Design System

A design system is ruleset governing the composition of a product. Meaning a blend of framework , components and UI toolkits.

Theory

Design systems are largely used by big companies to define company guidelines around design. Knowing what rules to follow from the beggining of a software product design wise can shorten design implementation a lot

Getting Started

Style pattern

To begin we have to break a design into it`s components.

For example the heading title carries a collection of styles like :

  • Typeface
  • Weight
  • Letter Spacing
  • Size
  • Color

These properties needs to be broken down to their bare-bone form. These will be the values that will define our global style pattern than can be used to design and build every aspect of our products. This eliminates uncertainties on the developers side.

Colours

As we are building software for different customers we will have to study the colors and company culture and mission or any other design languages. The most obvious style that can be choosed is the colour. We need to establish the primary colour and a secondary colour usually a complementary one or accent color.

Second colours to be taken in consideration are Success and Failure colours. Most of the time Green and Red.

Example of colour usage:

At the end we need grey colors. Modern UI uses at least 3 grey colours.

  • Light grey for background (#f1f1f1)
  • Slight darker grey for borders
  • Medium grey for font , subheading
  • Dark grey for headings , backgrounds, body

More grey variations can be used , different stroke or border shades etc. But the point is to define them from the start.

We also need tint and shades of the colours we choosed. This is mainly because we can use them to create light backgrounds , dark strokes, add depth to elements.

Shadows

Shadows are commonly used among most of UIs. Shadows are used to create perspective to components and even lift effects. 4 Shadow types inside the app should be enough to cover component needs.

  • Subtle Shadow
  • Hover effect shadow
  • Strong shadow for popovers or dropdowns
  • Distant shadow for modals

Animation simulation. Hovering the mouse on the container at the bottom results in an lift animation as in the container above.

Typography

To obtain a visual hierarchy inside the app we need to devine various font sizes and a font family for the entire app. Although 1 font family should be sufficient for an app a maximum of two fonts can be used on the app.

First step is to select the font family and then the needed headings. In theory "em" unit is the new standard for font sizes but in practice percentage has proven to be more efficient for scaling.

Border radius

Border radius is used among different components from buttons to containers to modals. So it`s best to adopt 3 border size units . 2px , 4px and 7px.

  • Small border radius can be used for checkboxes and labels.
  • Medium border radius for buttons and inputs
  • Large border radius for cards , modals and containers

Scaling - grid alignment.

For spacing and scaling please refer to Next Chapter - Grid.

Component library

After defining a global style , using this we can start building a component library , mapping styles to components.

Defining the component library for global use is essential rather than while developing having to modify a component in a specific part of the product.

Avoid overriding a global component because it can erode consistency.

Button

Example of building a button component (Using only predefined styles in the above part.) and applying the styles

Small border radius (2px) , Medium font ( 1.25em) , Height (Grid article - 32px) , Small Shadow (Subtle shadow) , Background color (Primary company color)

Simply put we will no longer need to overthink each aspect during developing rather we pick defined elements and add them to an component.

Easy component building

Repeating the button example simply plucking values from the style pallete we can define beautiful and consistent components.

Dropdown - Composed of a container with a Strong shadow A light colored border (defined in the first chapter) 3 Font sizes - 1.25 em for title , 0.8em for "view profile text" , and 1 em for the list elements and hover effect obtained from a shade from the primary color.

Using this method we can design an entire component library.

Conclusions

Components should only have defined the styles that will appear uniform in every instance. Do not add margins directly to a component ( buttons , inputs , headings etc.) as this will differ from case to case .

There will components that you will not be defined and other times these values will be arbitrary and not reusable. But the keypoint is to reuse as much as possible.
