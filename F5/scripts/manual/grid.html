The 8pt Grid

The Grid

This grid guide is meant to help designers and developers to have a common language with the purpose of laying out UI designs quick and consisten. It is very helpful for mobile designs.

Theory

App designs (prototypes and design files) is the main way to portray how the UI could like like and it`s the easiest way to communicate a proposal to the clients and developers, but in the end only what the developers can reproduce in code will make it to productions. There are severals concerns regarding the fact that even though tehnology allows the developers to fully implement almost everything from a design file in most cases it doesn`t. And this is caused by reasons like : - deadlines , the design might not be practical to build , developer prioritising functionality rather than design work.

Getting Started

Box Model

Box model consists of 4 elements : margin , border , padding and element size.

Border: the thickness of the stroke around the edges of an element. Most design tools do not allow this to affect overall spacing and dimensions.

Padding: the space between the bounds of an element and its child elements

Margin: the space between the bounds of an element and neighboring objects

Principles

The 8pt grid consists of placing elements while maintaining a 8pt increment in sizing and othe box model elements.

The point (pt) is a measurement unit for space that depends on screen resolutions. For example at @1X resolution 1pt = 1px.

@2px 1pt = 4px (because 2x means double resolution)

Multiples of 8 are used to establish sizing around an app. Sizing like margin , padding , borders and element size.

Typography

Most of the time text size and line-height will not conform to the UI grid . To align text to the UI grid combine the 8pt grid with a 4pt grid . Set the text to a consistent size then use padding to create the block element in such a way that is divisible by 8.

Method

There are two ways to apply the 8 pt grid. The Hard Grid method (incrementing everything in a 8point) and a Soft Grid method that measures 8point increment between individual elements. Most of the times the soft grid is used because it`s faster to implement and you can reach a high quality without spending additional time on it.

Why use the 8pt Grid

Using an even number will ensure scaling for a wide variety of devices. The majority of screen sizes are divisible by 8.

Value

When scaling follow a certain rule the result is a more consistent UI.

For designers: Efficient , less time spend on decisions on how to mantain consistency between elements

For developers: This eases communication between designer and developer , eliminating friction related to the "pixel perfect". Developers can easily identify 8 pixels increments instead of constant need for measurements.

For end client: Consistent UI and experience.
